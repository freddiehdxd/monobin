# AGENTS.md

Monobin — a single-binary Go SSR + islands framework. **Extend & reconcile; don't rewrite working code.**

## Commands
- `go test ./...` · `go vet ./...` — verify
- `go build -o monobin .` — build (NOT `-o app`; `app/` is a directory)
- `monobin check` — static validation; run before commit/CI (exits non-zero on error)
- `monobin routes [--json]` — list routes
- `cd islands && npm run build` — rebuild client assets into `app/assets/`
- `go run . dev` — dev server (reads `app/` from disk; run from the repo root)

## Non-obvious patterns
- **Go core is standard-library only.** No web framework / router / session libs. Ask before adding ANY Go dependency.
- **Islands must be registered** in `islands/src/entry.js` (the `islands = {}` map) or they silently don't hydrate. `monobin check` catches this.
- **Tailwind `@source`**: classes used only in `app/**.html` get purged unless `islands/src/style.css` keeps `@source "../../app"`.
- **`[slug].html` templates** are read with `fs.ReadFile` + `template.Parse`, never `ParseFS`/`ParseGlob` (the `[ ]` are glob metacharacters).
- **Middleware & auth are user-land** (`app.Use`, `examples/auth`) — never added to `framework/`. Auth is SSR-only; mark gated routes `app.NoStatic(pattern)` so `build` skips them.

## Boundaries
- Don't edit `app/assets/*` — generated by Vite (`npm run build`).
- Keep the framework small; one way to do each thing.
